This implementation uses the data.json file to store the customer and order data. The first route /customers returns the entire collection of customers. The second route /customers/:id returns a single customer record corresponding to the provided id parameter. The third route /customers/:id/orders returns a collection of orders for the customer with the provided id parameter. Finally, the fourth route /customers/:id/orders/:orderId returns a single order record for the customer with the provided id parameter and orderId parameter.
